Background 
Data analysts across every industry follow structured, repeatable workflows: acquire data, validate and clean it, 
engineer features, apply analytical models, validate results, and communicate insights through reports and 
visualisations. These workflows are perfect candidates for LLM skill authoring — because the logic is well-defined, 
the computations are deterministic, and the outputs must be precise and auditable. 
In this assignment you will author a production-quality skill package that instructs an LLM to perform a complete 
data analytics workflow for a specific business domain. The skill takes a real-world dataset (from Kaggle, UCI ML 
Repository, government open-data portals, or a synthetic generator you build) along with user configuration inputs, 
and orchestrates the full pipeline: data validation, feature engineering, modelling, result validation, segment/insight 
interpretation, and professional report generation. 
What the Skill Expects at Runtime 
Every analytics skill must accept two categories of input: 
1. Dataset Input 
A CSV, Excel, or Parquet file containing the raw data. The SKILL.md must specify the minimum required columns (with 
types and constraints), how to handle optional columns, and validation rules to reject malformed data early. The dataset 
can come from Kaggle, a public API, or a synthetic data generator included in your scripts/ folder. 
2. User Configuration Inputs 
Domain-specific parameters the user provides to customise the analysis — e.g., number of segments desired, date range 
filters, target metric, confidence level, business constraints, or industry context. The SKILL.md must list all parameters with 
types, defaults, valid ranges, and examples. 
The skill must validate both inputs before proceeding: check column existence, data types, null percentages, date 
ranges, and parameter bounds. If validation fails, the skill produces a clear error report explaining what’s wrong and 
how to fix it — not a cryptic crash. 
Mandatory Analytics Pipeline Stages 
Your SKILL.md must guide the LLM through all six stages of the analytics pipeline. Each stage must be a distinct, 
documented step with defined inputs, outputs, and validation checks. 
1. Data Validation & Profiling 
Before any analysis, profile the incoming dataset: row/column counts, data types, null percentages, duplicates, value 
distributions, and outlier detection. Produce a data quality summary. Reject or warn if quality thresholds are 
violated (e.g., >30% nulls in a critical column, date column not parseable). 
2. Data Preparation & Feature Engineering 
Transform raw data into analysis-ready features. This is domain-specific: for RFM it means computing 
Recency/Frequency/Monetary from transaction logs; for demand forecasting it means lag features and rolling

averages; for marketing mix it means ad-stock transformations. Document every feature, its formula, and its 
business meaning in REFERENCE.md. 
3. Modelling & Analysis 
Apply the core analytical method — segmentation, regression, optimisation, forecasting, etc. The skill must run at 
least two alternative approaches (e.g., K-Means vs. Hierarchical clustering, or ARIMA vs. Exponential Smoothing) so 
the results can be compared. Document algorithm choice rationale, hyperparameters, and assumptions. 
4. Model/Result Validation 
Apply quantitative validation techniques to choose the best result: silhouette scores and elbow method for 
clustering, cross-validation metrics for regression, backtesting for optimisation. The skill must not just pick a result 
— it must justify the choice with numbers. 
5. Insight Generation & Interpretation 
Translate analytical outputs into business language. The LLM must explain what the segments mean, what the 
forecast implies for planning, what the optimal allocation achieves. This is where domain expertise in 
REFERENCE.md pays off — the skill should reference industry benchmarks, typical ranges, and actionable 
recommendations. 
6. Report Generation 
Produce a professional, multi-section report (HTML with charts, or PDF/DOCX) containing: executive summary, 
methodology description, data quality summary, analysis results with visualisations, validation metrics, business 
interpretation, recommendations, assumptions & limitations, and a data appendix. 
7. No Shortcuts 
Each stage must be a real, substantive step in your SKILL.md — not a hand-wave like “now analyse the data.” The LLM 
must know exactly what to compute, what script to call, what output to expect, and what validation to perform before 
moving to the next stage.

Detailed Walkthrough: RFM Segmentation Skill (this is sample) 
To make expectations concrete, here is a step-by-step walkthrough of how a well-authored RFM Segmentation skill 
would work. Use this as a reference for the depth and specificity expected in your own skill. Do not choose this skill 
as this is used for demonstration here. 
An example RFM Segmentation — End-to-End Pipeline 
Step 1: Input Collection & Dataset Validation 
Accept a transaction CSV with required columns: customer_id, transaction_date, amount. Optional: 
product_category, channel. User inputs: analysis date (default: max date + 1 day), number of segments (default: 
4–6). Validate: no nulls in required columns, dates parseable, amounts positive, ≥100 unique customers, ≥6 
months of history. 
Step 2: Data Profiling & Quality Report 
 
Page 9 
Compute: total transactions, date range, unique customers, average transactions per customer, amount 
distribution (min/max/mean/median/std), null counts, duplicate check. Produce data quality summary. Flag 
anomalies (e.g., single-transaction customers >50%). 
Step 3: RFM Feature Engineering 
Aggregate per customer: Recency = days since last purchase; Frequency = transaction count; Monetary = total 
spend. Log-transform if skewness > 2. Z-score standardise. Produce the RFM feature table. 
Step 4: Multi-Algorithm Segmentation 
Run K-Means for k = 3,4,5,6,7 with silhouette scores and inertia. Run Hierarchical Clustering (Ward) for same k 
values. Optionally run DBSCAN. Produce comparison table: algorithm × k → silhouette, Calinski-Harabasz. 
Step 5: Validation & Segment Selection 
Select best (algorithm, k) by silhouette + interpretability. Generate: elbow plot, silhouette plot, 2D PCA 
projection. Verify no segment <5% or >40% of customers. 
Step 6: Segment Labelling & Profiling 
Per-segment: mean R/F/M, customer count, % revenue, AOV. Assign labels from REFERENCE.md archetypes 
(Champions, At-Risk, Hibernating, etc.). Produce profile table with radar charts. 
Step 7: Actionable Recommendations 
Per segment: specific marketing actions from REFERENCE.md (e.g., Champions → loyalty upsell; At-Risk → win
back campaign). Include segment size, revenue at stake, suggested channels. 
Step 8: Professional Report Generation 
Compile into multi-section HTML: Executive Summary, Data Quality, RFM Distributions, Methodology & 
Validation, Segment Profiles (with charts), Recommendations, Limitations, Data Appendix. Include timestamp and 
parameters.

Choose a concrete analytics workflow from Customer Analytics, Marketing Analytics, Pricing Analytics, Supply Chain 
Analytics, HR Analytics, Financial Analytics, or Operations Analytics. The skill must encode the specific methodology

2. Real-World Dataset 
Use a publicly available dataset from Kaggle, UCI ML Repository, government open-data portals, or a domain
specific API. Alternatively, include a generate_synthetic_data.py script that creates realistic synthetic data with 
known ground-truth properties. Document the data source, license, column definitions, and any preprocessing in 
data/data_dictionary.md. 
3. SKILL.md with Full Analytics Pipeline 
Your SKILL.md must cover all six mandatory pipeline stages (Validation → Feature Engineering → Modelling → 
Validation → Interpretation → Reporting). Each stage must specify: what to compute, which script to call, what 
output to expect, what validation to perform, and what to do if something fails. 
4. REFERENCE.md with Domain Knowledge 
Encode all domain-specific knowledge: feature formulas and their business meaning, algorithm selection criteria, 
segment archetype definitions, industry benchmarks, interpretation frameworks, and recommendation templates. 
The LLM consults this file — it must not hallucinate domain facts. 
5. Supporting Scripts (Minimum 3) 
Write at least three Python scripts covering: (a) data validation and profiling, (b) feature engineering and modelling, 
and (c) report generation with embedded visualisations. Each script must accept structured input (JSON/CLI args), 
return structured output, handle errors gracefully, and include docstrings. 
6. Multi-Algorithm Comparison, if applicable 
The skill should run multiple analytical approaches and compare them quantitatively, when necessary. The LLM 
must not just pick the first result — it must evaluate, compare, and justify the final choice using appropriate metrics

7. Professional Report Output 
The final output must be a visually polished, multi-section report (HTML with embedded charts, or PDF). It must 
include: executive summary, methodology, data quality summary, results with visualisations, validation metrics,

Test the skill with at least three scenarios. This can be two separate datasets and instructions given to skill to run. Or

SKILL.md Writing Principles for Analytics Skills 
i. Specify Every Computation 
Don’t write “compute RFM features.” Write “For each customer_id, compute: Recency = (analysis_date − 
max(transaction_date)).days; Frequency = count of transactions; Monetary = sum of amount. Store as DataFrame 
with columns [customer_id, recency, frequency, monetary].” 
ii. Name Every Chart 
Don’t write “create a visualisation.” Write “Create a bar chart with segment names on x-axis, customer count on y
axis, bars coloured by segment label, value labels above each bar, title ‘Customer Distribution by Segment.’” 
iii. Define Validation Thresholds 
Don’t write “check if results are reasonable.” Write “Verify: silhouette score > 0.25 (acceptable), > 0.50 (good), > 
0.70 (excellent). If < 0.25, warn the user that segments may not be well-separated.” 
iv. Script Interfaces Must Be Exact 
Document the exact CLI command, input format, and output format for each script. Example: “Run: python 
scripts/run_models.py --input rfm_features.csv --algorithms kmeans,hierarchical --k-range 3,7 --output 
models_output.json” 
v. Separate Computation from Interpretation 
The modelling step produces numbers; the interpretation step gives them business meaning. Keep these separate in 
SKILL.md. The LLM should first generate results, then — in a distinct step — consult REFERENCE.md to interpret 
them. 
vi. Include a Report Template Description 
Specify the exact sections, order, heading text, and content requirements for the final report. The LLM should not 
improvise the report structure — it should follow a precise blueprint. 
Common Anti-Patterns for Analytics Skills 
✗. “Just Run K-Means” 
Running one algorithm with k=3 and calling it done. No validation, no comparison, no justification. This is not 
analysis — it’s a random number generator with labels. 
✗. Skipping Data Validation 
Page 11 
Feeding raw, uncleaned data into models. The LLM should catch nulls, duplicates, wrong types, and outliers before 
modelling — and report what it found. 
✗. Charts Without Context 
A scatter plot with no axis labels, title, legend, or explanation. Every visualisation must be self-contained.
